# PRODUCT

## 1. 产品概况

- 产品名称：Pan 私人云盘
- 平台：面向个人用户的 PC Web，并支持窄屏使用
- 当前阶段：MVP
- 产品目标：让用户安全地上传、整理、预览、下载、分享和回收个人文件。
- 主要用户：需要在浏览器中管理个人资料的注册用户；通过链接临时访问内容的分享访问者。
- 核心成功结果：用户可从注册登录开始，完整走通上传、文件管理、分享、删除与恢复闭环。

## 2. 范围

### 本期包含

- 邮箱注册、登录、修改密码和退出。
- 我的文件、文件夹层级与面包屑。
- 文件上传、下载，以及图片、PDF、纯文本、音频和视频的浏览器基础预览。
- 新建文件夹、重命名、移动、复制、删除，以及当前页面多选批量操作。
- 回收站、恢复、永久删除和一键清空。
- 只读链接分享、分享列表和取消分享。
- 名称搜索、排序、分页、列表/网格视图与空间用量。
- 默认按更新时间降序排列，文件夹与文件各自保持最新上传或修改的内容在前。
- 管理员创建轻量 API 应用，将指定账号的全部网盘资源开放给外部系统，并查看累计使用情况。

### 本期不包含

- 团队空间、复杂成员权限、文件版本和在线协作。
- Office 转换、OCR、视频转码、全文检索和智能分类。
- 企业审计、内容审核、多地域存储和对象存储。
- 找回密码、MFA、第三方登录和续期 Token。

### 约束与已确认决策

- 使用 Vue 3、TypeScript、Vite、FastAPI、PostgreSQL 与 Docker Compose。
- 网页会话使用 HttpOnly Cookie，写操作使用签名 CSRF Token 与同源校验；生产环境仅允许 HTTPS 发送会话 Cookie。
- 单用户默认容量 5 GiB，单文件上限 1 GiB；均可由环境变量调整。
- 回收站保留 30 天，分享默认有效 7 天。
- 实际文件保存在后端持久化目录，数据库只保存元数据。
- 同一目录内文件和文件夹名称按不区分大小写判重，不静默覆盖。

## 3. 用户与权限

| 角色 | 核心目标 | 可访问范围 | 关键限制 |
| --- | --- | --- | --- |
| 空间所有者 | 管理自己的文件 | 自己空间内的内容、分享与回收站 | 不能访问其他用户数据 |
| 分享访问者 | 查看或下载被分享内容 | 分享节点及文件夹后代 | 不能上传、修改、删除或二次分享 |
| API 应用 | 代表外部业务系统管理文件 | 管理员绑定的账号及允许操作 | 使用独立密钥，不能跨账号或越过操作权限 |
| 系统 | 维护生命周期 | 过期分享与回收站内容 | 不改变用户可见业务规则 |

## 4. 术语与业务实体

| 名称 | 定义 | 关键状态 | 与其他实体关系 |
| --- | --- | --- | --- |
| 用户 | 文件所有者和操作者 | active | 拥有一个空间 |
| 空间 | 用户文件与容量统计范围 | active | 包含唯一根目录和所有节点 |
| 节点 | 文件或文件夹的统一业务记录 | active / trashed | 通过 parent 形成目录树 |
| 文件对象 | 服务器实际保存的二进制内容 | available | 被文件节点引用 |
| 分享 | 只读访问凭证 | active / revoked / expired | 指向一个文件或文件夹节点 |
| API 应用 | 外部系统访问凭证 | active / disabled | 关联一个用户和操作权限 |

## 5. 模块与导航

### 模块树

```text
Pan
└─ 个人云盘
   ├─ 我的文件
   ├─ 我的分享
   ├─ 回收站
   └─ 账户与空间
```

### 导航模式

- 模式：任务型 PC Web 左侧一级导航；窄屏收为顶部抽屉。
- 选择理由：功能数量少但文件管理任务密集，侧栏能稳定承载目录主入口与容量状态。
- 默认入口：登录后进入根目录的“我的文件”。
- 返回与深链规则：文件夹 URL 保存 folder 参数；面包屑可返回任意上级；分享为公开深链。

## 7. 核心流程

### 注册与登录

1. 用户提交邮箱、显示名称和密码，系统校验并创建用户、空间和根目录。
2. 用户登录后由后端写入 HttpOnly 会话 Cookie；前端不接触登录 Token，并使用签名 CSRF Token 保护写操作。
3. 失败时保留非敏感输入并显示可操作错误。

### 浏览、创建与上传

1. 用户通过列表或面包屑选择目录。
2. 可创建文件夹或选择文件上传；前后端校验名称、重名、大小和容量。
3. 上传期间显示真实进度；成功刷新列表和容量，失败保留可重试任务。

### 管理文件

1. 用户从行操作重命名、移动、复制、预览、下载、分享或删除。
2. 后端校验所有权、节点状态、目标目录和重名规则。
3. 删除进入回收站；文件实际内容仍占容量。

### 回收与恢复

1. 用户查看回收站，可恢复或永久删除。
2. 恢复优先回到原目录，原目录失效则到根目录；重名自动生成恢复名称。
3. 永久删除清理文件内容并释放容量。

### 分享

1. 所有者选择节点和 1/7/30 天或永久有效期，创建随机分享链接。
2. 访问者只能浏览分享范围、预览和下载。
3. 分享到期、取消或源节点进入回收站后立即失效。

## 8. 页面状态

- 所有远程列表包含 loading、empty、error、success 与重试。
- 表单包含字段校验、提交中、防重复提交和服务端错误。
- 删除与永久删除说明影响范围并二次确认。
- 文件预览区分支持、不支持、加载失败和文件失效。
- 未登录请求统一返回登录；越权与不存在不泄露资源归属。

## 9. 数据与 API 能力

### 核心实体

| 实体 | 必要字段 | 创建者 | 可见范围 | 生命周期 |
| --- | --- | --- | --- | --- |
| User | email、name、password_hash、quota | 系统 | 本人 | 注册至删除账户 |
| Node | owner、parent、kind、name、size、storage_key、trashed_at | 用户/系统 | 所有者或有效分享 | 创建、回收、永久删除 |
| Share | token、node、expires_at、revoked_at | 所有者 | 所有者和持链接访问者 | 创建至过期/取消 |

### API 清单

| 能力 | 调用页面 / 操作 | 关键输入 | 关键输出 | 权限 | 主要错误 |
| --- | --- | --- | --- | --- | --- |
| 注册/登录/当前用户 | 认证与应用壳 | 账户信息 | HttpOnly 会话 / 用户 | 公开/登录 | 重名、凭据错误 |
| 浏览与面包屑 | 我的文件 | parent、搜索、排序、分页 | 节点、路径、容量 | 所有者 | 目录不存在 |
| 创建文件夹 | 我的文件 | parent、name | 文件夹 | 所有者 | 非法名称、重名 |
| 上传文件 | 我的文件 | parent、file | 文件节点 | 所有者 | 超限、重名、容量不足 |
| 节点操作 | 行操作 | node、name/target | 更新节点 | 所有者 | 冲突、循环移动 |
| 下载/预览 | 文件列表/分享 | node 或 token | 文件流 | 所有者/分享者 | 不支持、失效 |
| 回收站管理 | 回收站 | node | 恢复/删除结果 | 所有者 | 状态冲突 |
| 分享管理 | 我的文件/分享 | node、有效期 | 分享链接 | 所有者 | 源节点失效 |
| 开放文件接口 | 外部业务系统 | API Key、node、file | 节点或文件流 | API 应用 | 密钥、目录范围、权限、容量限制 |
